# Configuration

Run `python3 scripts/setup.py` for the normal Docker Compose setup. It creates a private `.env`,
generates two distinct internal service tokens, and asks for the required OpenAI key without
echoing it. Use `python3 scripts/setup.py --check` to validate presence and shape without making a
provider request or printing values.

`.env.example` contains the ordinary startup choices. `.env.advanced.example` contains the
supported operational overrides most deployments may need. Application defaults live in
`src/personal_rag/config.py`; Compose supplies production-safe networking and storage defaults.
Internal fixture flags such as `RAG_DEMO_MODE` are set by project scripts and are not deployment
configuration.

## Required values

| Setting | Purpose |
|---|---|
| `RAG_API_KEY` | Protects the application API; generated by the setup assistant |
| `RAG_QDRANT_API_KEY` | Protects Qdrant; generated separately |
| `RAG_OPENAI_API_KEY` | Required for answers and for default OpenAI embeddings |

Keep `.env` private. The application rejects placeholders, short internal tokens, and reuse of the
same token for the API and Qdrant in production.

## Provider profile

The default profile is OpenAI `text-embedding-3-large` at 3072 dimensions plus the configured
OpenAI answer model. Voyage embeddings use:

```dotenv
RAG_EMBEDDING_PROVIDER=voyage
RAG_EMBEDDING_MODEL=voyage-3-large
RAG_EMBEDDING_DIMENSIONS=1024
RAG_VOYAGE_API_KEY=your-voyage-key
```

OpenAI remains the answer provider, so `RAG_OPENAI_API_KEY` is still required.

Provider, embedding model, dimensions, parser version, chunk size, and overlap form an immutable
embedding profile. Changing one requires a new collection and deliberate reindex; follow the
[migration workflow](operations.md#embedding-profile-migration).

## Common host overrides

| Setting | Default | Meaning |
|---|---:|---|
| `RAG_API_PORT` | `8000` | Host-loopback API port |
| `RAG_UI_PORT` | `8501` | Host-loopback Streamlit port |
| `RAG_QDRANT_HOST_PORT` | `6333` | Optional host-loopback Qdrant port |
| `RAG_QDRANT_COLLECTION` | `personal_knowledge` | Vector collection name |
| `RAG_CHAT_MODEL` | project default | OpenAI answer model |

## Retrieval

- `RAG_RETRIEVAL_TOP_K` is the normal passage count.
- `RAG_RETRIEVAL_MAX_TOP_K` caps user/API choices.
- `RAG_RETRIEVAL_MIN_SCORE` controls deterministic low-support abstention.
- `RAG_MAX_QUERY_CHARACTERS` and `RAG_MAX_HISTORY_MESSAGES` bound requests and context.
- `RAG_CITATION_SNIPPET_CHARACTERS` bounds returned source passages.

Increasing retrieval can increase answer context and provider cost. It does not expand the
supported document set.

## Ingestion

- `RAG_CHUNK_SIZE` and `RAG_CHUNK_OVERLAP` control chunking and affect the immutable profile.
- `RAG_PARSER_VERSION` must change when parser behavior makes existing chunks incompatible.
- `RAG_UPLOAD_MAX_BYTES` is capped at 25 MiB by both API and UI.
- `RAG_MAX_PDF_PAGES` and `RAG_MAX_EXTRACTED_CHARACTERS` bound extraction.
- `RAG_UPLOAD_READ_BYTES` controls streaming reads, not the accepted file size.

## Reliability

- `RAG_JOB_MAX_ATTEMPTS`, `RAG_JOB_LEASE_SECONDS`, and `RAG_WORKER_STALE_SECONDS` govern durable
  crash recovery.
- `RAG_PROVIDER_TIMEOUT_SECONDS` and `RAG_PROVIDER_MAX_RETRIES` bound transient provider retries.
- `RAG_WORKER_POLL_SECONDS` controls queued-job pickup frequency.

Authentication, model, unsupported format, and size errors are terminal rather than blindly
retried.

## Operations and observability

- `RAG_LOG_LEVEL`, `RAG_JSON_LOGS`, and `RAG_METRICS_ENABLED` control sanitized telemetry.
- `RAG_DATA_DIR` owns SQLite metadata and retained uploads for host-run development.
- `RAG_QDRANT_MODE=http` is required in production so API and worker share one vector service.
- `RAG_CORS_ORIGINS` is a JSON list or comma-separated list of trusted UI origins.

Logs and metrics intentionally exclude questions, answers, document text, citation snippets,
embeddings, provider payloads, and secret values.
